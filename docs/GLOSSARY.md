# TSiJUKEBOX Glossary

Technical terms explained in simple language.

---

## A

### API (Application Programming Interface)
A way for different software programs to talk to each other. Think of it like a waiter in a restaurant - you tell the waiter what you want, and they bring it back from the kitchen.

### Authentication
The process of proving who you are, usually by entering a username and password. Like showing your ID to enter a building.

### Authorization
What you're allowed to do after you've logged in. An admin can do more things than a regular user.

---

## B

### Backend
The "behind the scenes" part of an application that you don't see. It handles data storage, user accounts, and processing.

### Browser
The program you use to visit websites, like Chrome, Firefox, or Safari.

### Buffer
Temporary storage for data. When music buffers, it's downloading a little bit ahead so playback doesn't stop.

---

## C

### Cache
Saved copies of data so it loads faster next time. Like keeping a photo on your phone instead of downloading it every time.

### Cloud
Storing data on remote servers accessed via the internet, instead of on your own computer.

### Component
A reusable piece of the user interface, like a button or a card that displays track information.

### Context
In React, a way to share data between components without passing it through every level.

---

## D

### Database
An organized collection of data, like a digital filing cabinet. TSiJUKEBOX uses this to store your music library, playlists, and settings.

### Deploy
To make an application available for use, like publishing a website.

### Dependencies
Other software packages that an application needs to work. Like ingredients in a recipe.

---

## E

### Edge Function
Small pieces of code that run on servers close to users, making things faster. Used for secure operations like talking to external services.

### Environment Variable
Configuration settings stored outside the code, like passwords and API keys. Keeps sensitive information safe.

---

## F

### Frontend
The part of an application that users see and interact with - the buttons, images, and text on the screen.

### Framework
A pre-built structure that helps developers build applications faster. React is a framework for building user interfaces.

---

## G

### Git
A system for tracking changes to code over time. Lets multiple people work on the same project.

### GitHub
A website for hosting Git projects and collaborating with others.

---

## H

### Hook
In React, a special function that lets you use features like state and effects in components.

### HTTP/HTTPS
The protocol used to transfer data on the web. HTTPS is the secure version (the 's' stands for secure).

---

## I

### IDE (Integrated Development Environment)
A program for writing code, like VS Code. It has features to help developers work faster.

### Interface
How users interact with software, or in programming, how different parts of code interact with each other.

---

## J

### JSON (JavaScript Object Notation)
A format for storing and exchanging data. Looks like this:
```json
{
  "name": "Song Title",
  "artist": "Artist Name"
}
```

### JWT (JSON Web Token)
A secure way to transmit information between parties. Used for authentication.

---

## K

### Kiosk Mode
A special display mode designed for public use, like in a bar or restaurant. Simplified interface with limited controls.

---

## L

### Library
A collection of pre-written code that developers can use. Also refers to your music collection in TSiJUKEBOX.

### Localhost
Your own computer when used as a server. The URL `localhost` points to your machine.

### Lyrics Sync
When lyrics appear on screen matching the timing of the music, like karaoke.

---

## M

### Metadata
Information about data. For music, this includes artist name, album, year, etc.

### Migration
Updating a database structure, like adding new columns to a table.

---

## N

### Node.js
A platform for running JavaScript outside a web browser. Used to build the backend of applications.

### npm (Node Package Manager)
A tool for installing and managing JavaScript packages (dependencies).

---

## O

### OAuth
A standard way for applications to access your accounts on other services (like Spotify) without sharing your password.

### Open Source
Software whose code is freely available for anyone to view, use, and modify.

---

## P

### Package
A bundle of code that can be installed and used in a project. Like a plugin.

### Playlist
A list of songs that play in order.

### PWA (Progressive Web App)
A website that can be installed on your device and works like a native app.

---

## Q

### Query
A request for information from a database or API. Like asking a question.

### Queue
The list of songs waiting to be played next.

---

## R

### React
A JavaScript library for building user interfaces. Created by Facebook.

### Repository (Repo)
A folder containing all the files for a project, including version history.

### REST API
A common way of designing APIs using standard web requests (GET, POST, etc.).

### RLS (Row Level Security)
Database feature that controls which rows of data each user can access.

---

## S

### Server
A computer that provides services to other computers. Hosts websites and applications.

### Session
A period of time when you're logged in. Ends when you log out or after inactivity.

### Spotify Connect
Feature that lets you control Spotify playback on different devices.

### State
The current condition of an application - what's playing, what settings are active, etc.

### Supabase
A backend service that provides database, authentication, and more. Like Firebase but open source.

---

## T

### Tailwind CSS
A utility-first CSS framework used for styling. Makes it easy to design without writing custom CSS.

### Terminal
A text-based interface for interacting with your computer. Also called command line or console.

### Token
A piece of data used for authentication, like a digital key.

### TypeScript
A programming language that adds type checking to JavaScript, helping catch errors early.

---

## U

### UI (User Interface)
The visual elements users interact with - buttons, menus, screens.

### UX (User Experience)
How easy and pleasant it is to use an application.

### URL (Uniform Resource Locator)
A web address, like `https://example.com`.

---

## V

### Vite
A fast build tool for web applications. Pronounced "veet" (French for "fast").

### Visualizer
Animated graphics that respond to music, creating visual effects.

---

## W

### WebSocket
A technology for real-time communication between browser and server. Used for live updates.

### Widget
A small component that displays information, like the weather widget.

---

## Y

### YAML
A format for configuration files. Similar to JSON but uses indentation instead of brackets.

---

## Z

### Zod
A library for validating data in TypeScript. Ensures data is in the correct format.

---
